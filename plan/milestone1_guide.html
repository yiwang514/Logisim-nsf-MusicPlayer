0先回答你的问题：建几个 .circ？用哪些部件？

结论：三个 .circ，一人一个；本里程碑在 apu.circ 里干活

你既然还要带队友过答辩，多 .circ 正好——一人守一个文件、答辩各讲自己那块，互不踩脚。按计划书分工拆三个文件：

文件负责人里面装什么
top.circA · 组长main 顶层 + 控制面板 + 计时器；把 cpu/apu 当“库”加载进来接线
cpu.circC · CPUCPU 五段 + ALU + RegisterFile（R0–R3）
apu.circD · APU4 通道 + 频率换算 + $4015 使能 + 4 Buzzer；本里程碑从这里起步

三个文件靠 Logisim 的“库”机制拼起来：在 top.circ 里用 项目 → 加载库 → Logisim-evolution 库…（Project → Load Library → Logisim-evolution Library…） 选中 cpu.circ、apu.circ，它们就会出现在左侧浏览器里、能像普通部件一样拖出来实例化。只要每个文件对外的输入/输出 Pin 不变，内部怎么改都不影响别人——这就是多 circ 协作的关键，也是答辩时“我负责这个文件，对外接口就这几个 Pin”的天然边界。

本里程碑（= D · APU 的开头）只新建并改 apu.circ
apu.circ                ← D 组的文件，本里程碑新建它
├── BuzzerStress        ← 任务 A：声音压测（用完留作回归测试）
└── PulseOneShot        ← 任务 B：单通道点亮（弹出 A4）
   （后续里程碑再往这个文件加 Channel_Pulse / Channel_Tri / Channel_Noise / APU 顶层）

top.circ 和 cpu.circ 本里程碑先不用建——等做到 CPU / 顶层时再开新文件。

1Logisim 基本操作速成（脚趾头级）

下面 6 个动作覆盖本里程碑全部操作。记住它们，后面不再重复解释。

动作怎么做
选/移动 部件点工具栏最左的箭头＝编辑工具（Edit Tool）。点中部件可拖动；空白处框选多个。
放置部件在左侧浏览器里展开某个库（如 Wiring、Memory），点一下要的部件，再到画布上点一下放下。连续点可放多个；按 Esc 退出放置。
改属性用箭头工具选中一个部件，左下角“属性表（Attribute Table）”会列出它的全部属性，直接在那里改数值（位宽、Value、Waveform 等）。
连线还是箭头工具：把鼠标移到某个部件的引脚端点（小圆点），光标变成十字，按住拖到另一个端点松手即成一根线。线会自动走直角。
拨动/查看值（Poke）点工具栏的手指图标＝手形工具（Poke Tool）。点输入 Pin 可改它的值；点 Clock 可手动打一拍；点 ROM 可翻看内容。
给部件贴标签选中部件 → 属性表里的 Label 填名字（如 FREQ）。纯为可读，强烈建议给每个 Pin/ROM 贴标签。
看懂线的颜色（调试全靠它）

亮绿 = 1（单 bit） 深绿 = 0 蓝 = 浮空/未驱动 红 = 冲突/错误 黑 = 多位总线 灰 = 宽度不匹配/未知

红线 = 两个输出顶牛（接错了）；灰线 = 两端位宽不一致（最常见的坑，比如把 14-bit 接到 8-bit）。看到红或灰先别往下做。

让仿真跑起来的三个开关（很多人卡在这）
  • 仿真 → 自动传播（Simulate → Auto-Propagate）：总开关，必须打勾。
  • 仿真 → 启用自动节拍（Simulate → Auto-Tick Enabled）：让 Clock 自动跳动。不打勾时钟不会动。
  • 仿真 → 自动节拍频率（Simulate → Auto-Tick Frequency）：选时钟快慢（Hz/kHz）。本里程碑选个中高速（如 1–4 KHz）来听效果。

2部件速查表：本里程碑要用的零件

“库”指左侧浏览器里的分类。所有数值都是已核实的（依据 Logisim 官方组件文档与本仓库 ref/logisim-evolution 源码）。

部件库关键属性 / 引脚本里程碑怎么用
蜂鸣器
Buzzer		输入/输出（扩展）
I/O Extra		 引脚：FREQ(14-bit)、ENABLE(1-bit)、VOL(默认7-bit)、PW占空比(8-bit)。属性：波形 Waveform、Frequency Measurement(Hz)、音量位宽 Volume Width。 声音出口。FREQ 必须在 20–20000 才出声；14-bit 上限 16383（我们的频率表已钳好）。
ROM存储器
Memory		 引脚：A(地址,西)、D(数据,东)、sel(南,1-bit)。属性：地址位宽 Address Bit Width(2–24)、数据位宽 Data Bit Width。右键 → 加载映像…（加载映像…（Load Image…）） 载入 v2.0 raw 文件。 当频率查找表。sel 单 ROM 时悬空即可（默认输出）。
计数器
Counter		存储器
Memory		 属性：Data Bits。引脚：时钟、输出 Q、（可能有）使能/载入/清零。 压测里自动扫频：每拍 +1，输出当 timer。
时钟
Clock		线路
Wiring		 1-bit 输出。靠「仿真（Simulate）」菜单的 启用自动节拍（Auto-Tick）和 自动节拍频率（Auto-Tick Frequency）驱动。 给 Counter 打拍。
常量
Constant		线路
Wiring		 属性：Data Bits、Value(十六进制)、Facing。 给 ENABLE/VOL/PW/timer 喂固定值。
引脚
Pin		线路
Wiring		 属性：Type(input/output)、Data Bits。 做可手动拨动的 timer 输入（用 Poke 改值）。
分线器
Splitter		线路
Wiring		 把多位总线拆成/拼成若干位段。属性：Bit Width In、Fan Out 等。 本里程碑用不到，列出备查（M2 拼 timer 用）。
探针 Probe / 16 进制数码显示
Hex Digit Display		线路 / 输入输出
Wiring / I/O		 把总线当前值显示出来（Probe 显示原始值，Hex Digit 显示十六进制数字）。 盯着 timer / 频率值，方便调试。
找不到某个脚？两招（强烈建议先用第 1 招）

① 把部件外观换成经典：选中部件 → 属性表 外观（Appearance） → 选 经典 Logisim（Classic Logisim）。默认的“演化/IEEE”大符号脚多、还带 IEC 注解（比如 ROM 右边那排 A 0…13 其实只是给一根 14 位总线 D 标的位编号，不是 14 个脚）。换经典后：
  • ROM = 简单方块：A(地址) 在左、D(数据，单根总线) 在右、sel 在下。
  • 计数器（Counter） = 时钟脚在底部带 ▷ 那个、D 在左、Q 在右。

② 鼠标悬停看 tooltip：把光标停在任意脚上，Logisim 会弹出它的名字。蜂鸣器（Buzzer）的 4 个输入脚就靠这个认（朝向一变方位就转，别死记）：

悬停提示脚位宽接什么
频率（frequency）FREQ14（固定）← freq ROM 的 D
启用声音（Enable sound）ENABLE1（固定）← 常量 1
音量（volume）VOL默认 7（由 音量位宽 Volume Width 定）← 常量 0x40
占空比（Duty cycle）PW8（固定）← 常量 0x80

默认朝向下 FREQ/ENABLE/VOL 在平边竖排（频率上·启用中·音量下），占空比在一个角上。想转方向用 朝向（Facing） 属性。

3任务 A · BuzzerStress：先把声音风险压掉

为什么第一件事是这个，而不是搭 CPU

核对过 Logisim 的 Buzzer.java 源码：Buzzer 每次输入变化都会重建 1 秒音频缓冲并新开一个音频 Clip，开发者自己注释警告“别接 4kHz”。真实音乐的颤音会让频率每秒变几十次 × 4 路。这条链路扛不扛得住，决定整套 APU 方案成不成立。所以先花 20 分钟用一个最小电路把它压测掉——若流畅则全线绿灯，若卡则我们改“按帧采样”节流。绝不能等 CPU 都搭完才发现声音垮。

这个电路长这样（数据流）

 Clock ──► Counter(11-bit) ──► ROM[freq_pulse] ──► Buzzer.FREQ
 (Simulate         每拍+1=timer    timer→真实Hz       Waveform=Square
  自动tick)                                            ▲   ▲   ▲
                                          ENABLE=1 ────┘   │   │
                                          VOL=0x40(7bit)───┘   │
                                          PW=0x80(8bit, 50%)───┘
A1

双击进入 BuzzerStress 子电路

确认左下角标题是 BuzzerStress 而不是 main。

A2

放 Clock

浏览器 线路 → 时钟（Wiring → Clock），画布左侧点一下放下。

A3

放 计数器（Counter），设 11 位

存储器 → 计数器（Memory → Counter） 放在 时钟（Clock）右边。选中它，属性表把 数据位宽 Data Bits 改为 11。

时钟脚在哪？在计数器左边那排控制脚里、带小三角 ▷、标着 2,3,5+/C6 的那一个（IEEE 符号；源码 Counter.java 里的 CK 脚，west 边）。把 时钟（Clock）的输出连到它。

要让它真的会数，还得：把 G5（计数使能 ct）接一个 线路 → 常量（Wiring → Constant）、位宽 1、值 = 1（否则时钟在跳、计数却不动）；M1 [load] 与 R 接 0 或不接。
嫌这个大符号难认？选中计数器 → 属性表 外观 Appearance 换成 Classic（经典），脚更少更直观，功能一样。

A4

放 ROM，设 11/14 位，载入频率表

存储器 → ROM（Memory → ROM） 放在 Counter 右边。属性表：地址位宽 Address Bit Width = 11，数据位宽 Data Bit Width = 14。
右键这个 ROM → 加载映像…（Load Image…） → 选 script/out/freq_pulse.txt。载入后 ROM 内部会显示一堆十六进制值（地址 0 处应是 3fff）。
把 Counter 的输出（11-bit）连到 ROM 的 A（西边）。

★ 加载映像时弹出「Hex 文件格式」对话框？只看顶部那一行

对话框顶部会写 decoded N of M words, K bits each，格式默认选 v2.0 raw（下面 words / auto / big-endian 默认勾选，都不用动）。关键看这行对不对：

  • 正确：decoded 2048 of 2048 words, 14 bits each → 直接点 确定。
  • 出错例（你刚遇到的）：decoded 2048 of 256 words, 8 bits each → 说明 ROM 位宽还没设对（还是默认 8 位地址 / 8 位数据）。它会把 14 位频率值砍成低 8 位（如 3fff→ff、3e6c→6c）、且只装前 256 个，音高全错。
    → 点 取消，先把 地址位宽=11、数据位宽=14 设好，再右键重新 加载映像（Load Image…）。
A5

放 Buzzer，设方波

输入/输出（扩展）→ 蜂鸣器（Input/Output Extra → Buzzer） 放在 ROM 右边。属性表：波形 Waveform = Square，Frequency Measurement = Hz（默认），音量位宽 Volume Width 保持 7。
把 ROM 的 D（东边，14-bit）连到 Buzzer 的 FREQ 脚。两端都是 14 位，线应是黑色总线，不是灰色——灰了说明位宽没对上，回去检查。

A6

给 Buzzer 的另外三个脚喂常量

放三个 线路 → 常量（Wiring → Constant）：
• ENABLE：Data Bits=1，Value=1 → 接 Buzzer 的 ENABLE 脚（中间那个 1-bit 脚）。
• VOL：Data Bits=7，Value=40（十六进制，=64，约半音量）→ 接 VOL 脚。
• PW：Data Bits=8，Value=80（=128，占空比 50%）→ 接 PW 脚。

A7

开仿真，听！

依次：仿真 → 自动传播（Simulate → Auto-Propagate）✔ → 启用自动节拍（Auto-Tick Enabled）✔ → 自动节拍频率（Auto-Tick Frequency） 先选 1 KHz。
Counter 会不停 +1，timer 0→2047 循环，频率随之扫动，你应当听到一个连续上下扫的音调。

判定标准（这就是本任务的目的）
  • 绿灯：扫频音连续、不爆、不卡顿。逐步把 Tick Frequency 调到 2K/4K 再听。能撑住 → APU 方案稳，放心往下做。
  • 黄灯：低速流畅、高速开始爆音/卡顿。→ 记下临界频率；后续给 Buzzer 输入加“按帧采样”寄存器（约 60Hz 更新），把变化率压下来。
  • 红灯：几乎不出声或严重卡死。→ 先别搭 4 通道，回来找我，我们改方案（节流 / 降颤音）。
A8

（可选但强烈建议）压测 4 路同响

把 ROM→Buzzer 那串复制成 4 份（4 个 Buzzer，可共用同一个 Counter/ROM，也可各接不同 Counter 让频率不同）。同时开声，验证多 Buzzer 一起响时声卡能否自动混音且不卡。这直接预演真实 4 声道场景。

4任务 B · PulseOneShot：用频率表弹出 A4 和音阶

任务 A 证明“频率→声音”能跑。任务 B 把它变成可控音高：你给一个 timer 值，它发出对应的真实 NES 音高。这就是单个 Pulse 通道的核心（暂不接寄存器拆分，纯手动喂 timer，验证音高正确）。

数据流

 Pin[timer] (11-bit, 用Poke改值) ──► ROM[freq_pulse] ──► Buzzer.FREQ
                                                          Waveform=Square
   ENABLE=1 / VOL=0x40 / PW=0x80 ──────────────────────► (同任务A)
 （可选）Pin 上再接一个 Probe，随时看你拨的 timer 值
B1

进入 PulseOneShot

B2

放一个输入 Pin 当 timer

线路 → 引脚（Wiring → Pin）。选中它：Type=Input，Data Bits=11，Label=timer。

B3

放 ROM（同任务 A 的设置）

地址位宽 Address Bit Width=11，数据位宽 Data Bit Width=14，务必先设好位宽再右键 加载映像…（Load Image…） → script/out/freq_pulse.txt（加载对话框若显示 256 words / 8 bits 就是位宽没设，见任务 A 那个红框）。
Pin timer → ROM 的 A。

B4

放 Buzzer + 三个常量（同任务 A 的 A5、A6）

Waveform=Square；ROM D(14)→Buzzer FREQ；ENABLE=1、VOL=0x40、PW=0x80。

B5

开仿真，拨 timer = 253，听 A4

开 自动传播（Auto-Propagate）（这次不需要自动节拍（Auto-Tick），没有时钟）。切到 Poke 工具（手指），点 timer 那个 Pin，输入十进制 253（或十六进制 FD）。
频率表里 freq_pulse[253] = 440 Hz = 标准音 A4，你应当听到一个稳定的 440Hz 方波。

B6

弹个音阶证明音高可控

继续用 Poke 改 timer，验证“数值越小、音越高”。参考几个值（Pulse，NTSC）：

timer≈频率≈音
253440 HzA4
190≈586 Hz≈D5
126≈881 Hz≈A5（高八度）
63≈1748 Hz≈A6

能听出音阶关系 = 频率表 + Buzzer 这条链路音高正确。

小知识：为什么不用接 Clock

Buzzer 的音高来自 FREQ 引脚的数值，由它内部的音频线程按真实时间发声，跟 Logisim 时钟无关。所以静态喂一个 timer 就能持续出声。Logisim 时钟只在“需要随时间变化”（计数、CPU 走指令）时才用到。

5里程碑 1 验收清单

  • 新建了 apu.circ 文件，里面有 BuzzerStress 与 PulseOneShot 两个子电路。
  • BuzzerStress：Counter 自动扫频，能听到连续扫频音；记录下它在 1K/2K/4K Tick 下是否流畅（这条结论决定后续要不要节流）。
  • （建议）4 个 Buzzer 同时响过一次，确认多路混音可用。
  • PulseOneShot：timer=253 时听到稳定 A4(440Hz)。
  • 改几个 timer 值能听出音阶（越小越高），确认音高映射正确。
  • 所有线都是绿/黑，没有红线或灰线。
  • 已存盘并备份一份 apu_m1.circ。
做完这一步，你已经验证了什么

① 频率换算表 ROM 正确（音高对）；② Buzzer 出声链路可用；③ 最关键的——Buzzer 在频率快速变化下扛不扛得住（整套 APU 方案的命门）。这三件事一旦绿灯，后面就是按部就班把寄存器译码、四通道、CPU 一块块接上去。
