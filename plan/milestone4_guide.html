★ 实搭避坑补充（M4 已用「脚本生成整套电路 + 无界面点测」跑通验证；以下是手搭时易踩、原文没点透的点）
  • 两个译码器（Decoder）别搞混：opcode 译码用里程碑 3 那个 选择位数（Select Bits）=4（16 路）译码器（Decoder），直接取它的第 4/5/6/7 路当 is_loadi/is_add/is_sub/is_writereg；而写回用的「Rd→片选」是另一个新的 选择位数（Select Bits）=2（4 路）译码器（Decoder）。两者选择位数、输出根数都不同，别共用。
  • 标签唯一性同样卡 R0–R3：4 个 寄存器（Register） 标签（Label） 叫 R0–R3，那么对外观测脚就得叫 r0_out 之类，不能也叫 R0（否则加载时标签被清，见里程碑 3 同款陷阱 Circuit.isExistingLabel()，不分大小写）。
  • WRITEREG 要同时改 value 和 WR 两处：APU 的 value 不再直接接 instr[7:0]，而要先过一个 is_writereg ? R[Rs] : instr[7:0] 的 多路复用器（Multiplexer）；写脉冲 WR 也要对 is_write 或 is_writereg 都发（即 (is_write∨is_writereg)·clk）。漏掉任一处，WRITEREG 都写不进 APU。
  • 比较器（Comparator）数值类型：快档钳位比 imm 与 delta，二者都是小正数（<0x8000），无符号（Unsigned）与补码（2's Complement）结果相同，默认即可；要严谨就显式设无符号。
  • 变速实测数值（delta=8）可当自检标准：normal 装 imm、slow 装 imm+delta、fast 装 imm−delta 且钳到 ≥1——实测 WAIT 16 三档分别装 16 / 24 / 8，WAIT 5 快档钳到 1（没有回绕成 65533）。
  • CPU→APU 接线可「靠位宽自检」：APU 五个输入口位宽各不相同（ch_mask=4、WR=1、reg_id=4、value=8、frame_clk=1）。接错顺序必触发位宽不匹配（灰线）。连完只要全程无灰线，顺序基本就对了。

0总览：ALU 在哪、数据流

里程碑 3 的 CPU 里，WAIT 指令把立即数 imm 直接装进倒计时计数器。本里程碑在这条装载路径上插一个 ALU：装进去的不再是 imm，而是 有效等待 = imm 经加/减调速后的值。于是控制面板按一下「慢/快」，真实曲子的节奏就变——这就是「可控加减运算参与播放主功能」，不是摆设。

里程碑 3:   IMM(12) ──位扩展──► WAIT 计数器.数据脚
里程碑 4:   IMM(12) ──位扩展──►┌───────── ALU 调速 ─────────┐──► WAIT 计数器.数据脚
                               │  normal: 有效 = imm          │
            tempo_delta ──────►│  slow  : 有效 = imm + delta  │  (加法器)
            tempo_mode(2) ────►│  fast  : 有效 = imm − delta  │  (减法器，且钳到 ≥1)
                               └──── 多路复用器 按 tempo_mode 选 ┘

另一处用同一类 ALU：通用寄存器 R0–R3 + 指令 LOADI/ADD/SUB/WRITEREG，配 数码显示 现场证明 R2 = R0 + R1。
控制面板(top.circ)：播放/暂停(门控时钟) · 停止(复位+reg9=0) · 上/下一首(曲号计数器 → N:1 MUX 选 ROM；4 首=2 位曲号+4:1) · 4 通道开关(ch_mask)+使能探针。
（不做 MM:SS 实时计时：Buzzer 出声靠真实墙钟 Hz、CPU 节拍又是分频来的，二者对齐到"真实秒"既不稳也无演示价值，故略去。）
两处 ALU 是否要合一？

detailed_plan §5 的本意是「一个 ALU，主用途接 WAIT 变速、同时拿 R0–R3 demo 做可视化」。合一更优雅(输入再加一层 MUX 在 imm/delta 与 Ra/Rb 间选)，但对单人施工分开搭更清楚、也都能过验收：§2 的加/减专管播放调速(已满足「参与播放」)，§3 的 ALU 专做寄存器运算 demo。本指引按「分开」写，最后给一个「合一」的提示。

2★ ALU 接入 WAIT 变速播放 必做 · 老师硬性要求

目标：控制面板给一个 2 位 tempo_mode(0=正常/1=慢/2=快) 和一个 tempo_delta(调速幅度，比如 8)，让每一条 WAIT 装载前先经 ALU 算 有效等待。这样真实播放时加/减器每帧都在工作。

A1

准备 tempo_delta 和 tempo_mode 两个输入

本节先在 CPU 里把它们当输入 引脚（Pin）引进来（§4 控制面板再接到真正的开关）：tempo_delta(16 位，先接个 值=0x08 的 常量（Constant） 测试)、tempo_mode(2 位)。delta 用 16 位是为了和 WAIT 计数器(16 位)对齐。

A2

放 加法器（Adder） 和 减法器（Subtractor），各 16 位

都设 数据位宽（Data Bits）=16。接线：

  • 加法器：a=imm(经里程碑 3 的位扩展器补成 16 位)、b=tempo_delta、进位输入接 值=0 的 1 位 常量（Constant）。输出 和 = imm+delta(慢)。
  • 减法器：被减数=imm(同一根 16 位)、减数=tempo_delta、借位输入接 0 常量。输出 差 = imm−delta(快)。
A3

快速档下限保护（比较器（Comparator） + 一个 2:1 MUX）

imm − delta 在 imm ≤ delta 时会变成负数、按无符号回绕成接近 65535 的巨值 → 卡死。修法：放一个 比较器（Comparator）(数值类型=无符号（Unsigned）)，a=imm、b=tempo_delta；放一个 多路复用器（Multiplexer）(选择位数=1、数据位宽=16)：
• 选择脚 = 比较器的 < 或 =（即 imm ≤ delta，用一个 或门（OR Gate） 把 < 和 = 并起来）。
• 数据 0 = 减法器的 差；数据 1 = 值=0x0001 的 16 位 常量（Constant）。
这样快速档真正用的是 钳位后的快速值(下限 1)。

A4

调速三选一：一个 多路复用器（Multiplexer） 选 有效等待

放 多路复用器（Multiplexer）：选择位数（Select Bits）=2、数据位宽（Data Bits）=16。选择脚接 tempo_mode(2 位)。数据脚：

tempo_mode该路数据含义
0 正常imm(16 位)原速
1 慢加法器 和 = imm+delta等待更久 ⇒ 更慢
2 快A3 钳位后的 imm−delta等待更短 ⇒ 更快
3 (未用)随意(接 imm)—

把这个 MUX 的输出，接到里程碑 3 那个 WAIT 计数器的数据脚(替换原来直连的 imm)。完工：每条 WAIT 现在都经 ALU。

★ 关于「微调播放速度」：粒度、怎么调到刚好、与 4kHz 测试是什么关系

这套 ALU 调速本身就是"微调"工具——关键是 tempo_delta 别设大（A1 举例 8 只是为了演示明显）。每帧只有一条 WAIT，整帧 = frame_cycles(=32) 拍；慢档给这帧 +delta 拍 ⇒ 该帧变 32+delta 拍、速度 ×32/(32+delta)：

tempo_delta慢档(mode=1)快档(mode=2)每档约
1×32/33 ≈ −3%×32/31 ≈ +3%~3% / 档
2≈ −6%≈ +7%~6%
8（A1 举例）≈ −20%≈ +27%大步，非微调

要"小幅微调"：把 tempo_delta 设成 1~2，每按一下慢/快只动 ~3~6%。想更细有两条路：① 把 tempo_delta 做成可调（§4 控制面板用一个 3~4 位 拨码开关（DIP Switch）/上下按钮给 delta 0~7，而非焊死常量）——慢/快 × delta 组合出一排细档；② 离线在 encode.py 把 frame_cycles 调大（如 64）⇒ 每档 ≈ delta/64 ≈ 1.5%/档，分辨率翻倍（代价：基准 CPU 时钟要相应调快；指令 ROM 几乎不变大，因为每帧仍只一条 WAIT）。

校准顺序 + 现状澄清（重要）

① 基准速度先靠 frame_cycles（离线）+ 自动节拍频率（Auto-Tick Frequency，运行时，但只有 1/2/4/8kHz 粗档）定到接近实时；② 微调再用本节 ALU 慢/快 + 小 delta 现场修零头。4kHz / frame_cycles=32 客观上只比实时快约 +4%（2000 拍 ÷ 32 = 62.5fps vs 真实 60.0988fps），不是 20%——之前听感偏快多半是被那串 ~60Hz 爆破"催"出来的；先听节流版再判断。真要抹平这 +4%：慢档 + delta=1（−3%）压到 ~+1%，或离线把 frame_cycles 改 33 重生成。
注意：平铺 ROM 试听台（apu_test）没有 CPU/WAIT，用不上这套 ALU 调速——它只能靠自动节拍频率（粗）或重生成 frame_cycles（细）。ALU 的运行时微调是"上了 CPU 之后"才有的能力，正是本里程碑交付的。

用点测环境验 ALU（不用肉眼盯波形）

把「imm / tempo_delta / tempo_mode → 有效等待」这块组合逻辑单独做成一个子电路、引出输入输出 引脚（Pin），写一个测试向量文件，用 --test-vector 跑：例如 imm=30,delta=8,mode=1 ⇒ 38、mode=2 ⇒ 22、imm=5,delta=8,mode=2 ⇒ 1(钳位生效)。一把就能确认加/减/钳位都对。命令与向量格式见 plan/logisim_test_harness.html §2。

顺带把里程碑 3 的「+2 拍」精确化（可选）

里程碑 3 提过 WAIT n 实占 n+2 拍。既然这条路上已经有减法器，可把装入值再 −2(并用 A3 同款钳位保证 ≥1)，得到精确 n 拍。但节奏本来靠节拍频率现场调，这步纯属锦上添花，不做也不影响验收。

3R0–R3 + LOADI/ADD/SUB/WRITEREG + 可视化 必做

这一节搭出「真正的 ALU + 通用寄存器」并用一段 demo 程序现场证明。指令位布局(取自 detailed_plan §3)：

op助记位布局 [15..0]语义

B1

放 4 个通用寄存器 R0–R3

存储器 → 寄存器（Memory → Register） 放 4 个，数据位宽（Data Bits）=8，标签 R0/R1/R2/R3。时钟统一接 CPU 主时钟，清除接 reset。(标签唯一性同里程碑 2：寄存器叫 R0，对外若要引出观测脚就叫 R0_out 之类。)

B2

读口：两个四选一 MUX 取出 R[Ra] 和 R[Rb]

放两个 多路复用器（Multiplexer）：选择位数=2、数据位宽=8。四个数据脚都接 R0/R1/R2/R3 的 Q。一个的选择脚接 Ra(=instr[9:8])、输出即 R[Ra]；另一个选 Rb(=instr[7:6])、输出 R[Rb]。
(用 Splitter（分线器） 从 INSTR 拆出 Ra=[9:8]、Rb=[7:6]、Rd=[11:10]，同里程碑 3 §3 的拆位手法。)

B3

ALU 本体：加法器 + 减法器 + 结果 MUX（8 位）

放 加法器（Adder） 与 减法器（Subtractor），数据位宽=8：a/被减数=R[Ra]，b/减数=R[Rb]，进/借位输入接 0。放一个结果 多路复用器（Multiplexer）(选择位数=1、数据位宽=8)：选择脚 = is_sub(opcode=0x6)，数据 0=加法和、数据 1=减法差。输出 = ALU 结果。

B4

写回：Rd 译码 + 写使能；LOADI 走另一路数据

每个寄存器的数据 D 该接什么？放一个写回 多路复用器（Multiplexer）(选择位数=1、数据位宽=8)：选择脚=is_loadi(opcode=0x4)，数据 0=ALU 结果(B3)、数据 1=imm8(=instr[7:0])。它的输出同时接到 R0–R3 全部的 D。
写使能：放一个 译码器（Decoder）(选择位数=2)，输入 Rd(=instr[11:10]) → 4 路 one-hot；第 k 路 与 是否“要写寄存器的指令”(=is_loadi 或 is_add 或 is_sub，用 或门（OR Gate）) 相与，接到 Rk 的写使能。于是只有 Rd 选中那个寄存器、且当前是 LOADI/ADD/SUB 时才在时钟沿写入。

opcode 译码复用里程碑 3

里程碑 3 已有一个 opcode → 16 路 译码器（Decoder）。is_loadi=第4路、is_add=第5路、is_sub=第6路、is_writereg=第7路，直接取来用。

B5

WRITEREG：把算出的值喂回 APU

里程碑 3 的 APU 写口 value 原本固定接 instr[7:0]。这里在它前面加一个 多路复用器（Multiplexer）(选择位数=1、数据位宽=8)：选择脚=is_writereg，数据 0=instr[7:0](普通 WRITE)、数据 1=R[Rs](再加一个四选一 MUX，选择脚=Rs=instr[7:6])。reg_id 仍取 instr[11:8]；写脉冲 WR 对 WRITE 和 WRITEREG 都要发(把 is_write 或 is_writereg 与时钟相与)。

B6

可视化 + demo 程序

给 R0–R3 的 Q 各贴一个 探针（Probe）(radix=16)，或用 16 进制数码显示（Hex Digit Display）(8 位要配 Splitter（分线器） + 两个数码显示)。
写一段 demo ROM 验证(右键 ROM 编辑内容…（Edit Contents…） 填这几个字)：

4005   LOADI R0, 0x05     ; R0 = 5
4403   LOADI R1, 0x03     ; R1 = 3
5840   ADD   R2, R0, R1   ; R2 = R0+R1 = 8
f000   END

单步/慢跑后 R2 应显示 8。改 ADD 为 SUB(把 5840 改 6840) 则 R2 = 5−3 = 2。这就是答辩用的「可控加减」现场证明。

4控制面板 / 多曲切换 / 通道开关 加分

这部分在 top.circ 里搭：用 项目 → 加载库 → Logisim-evolution 库…（Project → Load Library → Logisim-evolution Library…） 把 cpu.circ 和 apu.circ 当库加载进来，再围着它们搭面板。

★ 架构要点（实搭验证过）：多曲切换 = 把指令 ROM 移到 CPU 外、走总线（标准 CPU↔存储器分离）

原理（先讲清为什么必须这么做）：里程碑 3 的 CPU 把指令 ROM 内置了（PC→内部 ROM→译码），一颗 CPU 死绑一首曲子。要切歌，就得让多个 ROM 共用同一个 PC 地址、再用一个 多路复用器（Multiplexer） 选出"当前这首"的指令。这逼着你把 ROM 搬到 CPU 外面，让 CPU 对外只暴露两条总线：地址总线（PC 输出 → 喂给所有 ROM 的地址脚）和指令总线（选中那首 ROM 的数据 → 回灌 CPU）。这正是教科书的 CPU↔存储器分离：CPU 是边界清晰的运算核心、存储器挂在外部总线上。而"多个 ROM + 曲号选择"实质就是 NES 卡带的 mapper 换页（bank-switching）——非常真实，答辩时是个亮点。

具体做法：复制一份 CPU 子电路（取名 CPU_EXT 之类），把里面那颗指令 ROM 删掉，改成：① 给 PC 计数器（Counter） 的 计数值 Q 引一个输出 引脚（Pin） pcbus(16)；② 新增一个输入 引脚（Pin） instr(16)，接到原先 ROM 数据脚所驱动的那根 INSTR 线上。其余（拆位译码 / WAIT 停顿机 / ALU 变速 / R0–R3）原封不动。于是 CPU 对外接口变成：入 clk / reset / tempo_mode / tempo_delta / instr，出 pcbus / reg_id / value / wr / frame_commit。
（标签别撞名：输出脚叫 pcbus 而不是 pc，以免和内部 计数器（Counter）PC 重名被清，见里程碑 3 同款陷阱。）

top 里组装：加载库后放 CPU_EXT + N 个 ROM（N=本项目最终曲数 4；各 地址位宽=16/数据位宽=16，地址脚全接 pcbus 并联）+ 一个 多路复用器（Multiplexer，选择位数（Select Bits）=⌈log₂N⌉、数据位宽（Data Bits）=16，选择脚=曲号）→ 输出的 instr 回接 CPU 的 instr 入。4 首 ⇒ 选择位数=2、4:1 MUX、曲号是 2 位。

为什么不直接铺满 16 槽（踩过的坑）

早期版本固定铺 16 个 ROM（4 位曲号、16:1 MUX）。问题有二：① 16 个 4 万字的大 ROM 把 Logisim 拖得很卡、甚至影响后续曲目加载；② 实际只有几首，"下一首"会切到一长串空槽（静音）。所以最终按真实曲数 N 来定大小：曲号计数器位宽=⌈log₂N⌉、MUX 也只要 N 路。功能正确且轻量，远比塞满 16 首重要。

已验证参照：我按这套搭了 cpu_pureAI.circ（内含 CPU_EXT 子电路）+ top_pureAI.circ（用加载库装配两库、4 槽 ROM + 4:1 MUX、播放/暂停/停止/上下首/通道开关齐全），无界面实测 CPU_EXT 经总线取指正常、WAIT 16 精确停 18 拍。可对照着搭你自己的 cpu.circ/top.circ。

C1

播放 / 暂停（门控时钟）

放一个 D 触发器（D Flip-Flop）(存储器 → D 触发器) 当 running 标志，用 按钮（Button） 翻转它。CPU 主时钟 = 母时钟 与 running(一个 与门（AND Gate))：running=0 时 CPU 时钟不跳、PC 冻结、音乐停在当前音。

C2

停止

停止 = 复位 + 静音。复位用前面的 reset（PC→0，已和上/下一首并进同一根 rst）；静音不要靠"复位 APU"——APU 子电路没有 reset 输入口，reset 只复位 CPU、碰不到 APU 内部的 RegFile，所以单按复位会让上个音一直长响。正确做法：把"停止"并进 C4 的 ch_mask 静音门控（见下方 C4：active = run 与 非stop，停止时 ch_mask=0 直接静音）。这样不必给 APU 加复位、也不必单独清 reg9。

C3

上 / 下一首：曲号计数器 → N:1 MUX 选 ROM（4 首 = 2 位曲号 + 4:1）

原理：用一个计数器（Counter）当"曲号"，两个按钮（Button）一个让它 +1（下一首）、一个让它 −1（上一首）。计数器（Counter）的"方向"脚（Up/Down）就是干这个的——巧用它，不必两个计数器、也不必加减器。

  1. 两个按钮 → 一个时钟脉冲：放 下一首、上一首两个 按钮（Button）（隧道名 NEXTB/PREVB），用一个 或门（OR Gate） 把它俩并起来 → 接曲号计数器的时钟脚。含义："按了任意一个键，计数器就走一步"。
  2. 方向脚决定 +1 还是 −1：把计数器的方向（Up/Down）脚直接接 NEXTB。按"下一首"时 NEXTB=1 ⇒ 方向=递增 ⇒ +1；按"上一首"时是 PREVB 触发的时钟、而 NEXTB=0 ⇒ 方向=递减 ⇒ −1。
  3. 到头回绕：曲号计数器 数据位宽=⌈log₂N⌉（4 首 ⇒ 2 位）、最大值（Maximum Value）=N−1（4 首 ⇒ 3）、溢出时行为=回绕（Wrap around）。于是曲号在 0↔1↔2↔3 循环、自动绕回，不越界。计数值 Q（=曲号）引出隧道 TRACK。
  4. N 个 ROM + N:1 MUX：放 N 个 ROM（各 地址位宽=16、数据位宽=16），分别 加载映像…（Load Image…） track00.txt … track0(N-1).txt；所有 ROM 的地址脚都接 CPU 的 pcbus（并联读），N 个数据脚接一个 多路复用器（Multiplexer）（选择位数=⌈log₂N⌉、数据位宽=16），选择脚=TRACK，输出 = 当前曲的 instr 回接 CPU 的 instr 入。
  5. 切歌即复位（让新曲从头放）：把 CPU 的 reset 改成 手动复位按钮 或 切歌脉冲——即放一个 或门（OR Gate）：一个输入是原来的复位 引脚（Pin）/按钮（Button），另一个输入接 TRKCLK（=上一首∨下一首），输出才是真正接进 CPU 的 rst。这样按上/下一首的同一下，TRKCLK 拉高 → 既让曲号计数器 +1/−1、又异步清零 PC（WAIT/busy/R0–R3 一并复位），新曲从地址 0 干净开始。（别让复位引脚和或门同时驱动 rst 那根线——会变成"双驱动"冲突；正确做法是复位引脚先进或门、或门输出再叫 rst。）

本项目最终 4 首：slot 0=monitoring、1=ano_bando、2=悠久之风（FF3 #30）、3=sparkling_daydream。所以是 2 位曲号 + 4 个 ROM + 4:1 MUX（不要铺 16 槽，原因见上方架构框的警告）。

C4

4 通道开关（ch_mask）+ 实时使能探针

APU 子电路有一个 4 位输入口 ch_mask，每一位单独开/关一个声道（bit0=Pulse1、bit1=Pulse2、bit2=Triangle、bit3=Noise，位序与 NES $4015 一致，已对照 apu.circ 里 ch_mask 分线器（Splitter） 核实）。里程碑 3/4 调试时这里直接接 值=0xF 的 常量（Constant）（四路全开）；控制面板要把它换成四个可拨的开关，答辩时能现场单独静音某一路、对比音色。

搭法：放 4 个 1 位输入 引脚（Pin）（命名 ch_p1 / ch_p2 / ch_tri / ch_noise，可设 外观（Appearance） 当拨动开关用；把每个引脚的 初始值（Initial value） 设为 1，这样上电默认四路全开），各引出一根 CH0…CH3 线；用一个反向使用的 分线器（Splitter）（分支数（Fan Out）=4、输入位宽（Bit Width In）=4、各分支按位 0→0/1→1/2→2/3→3 映射）把 4 根单 bit 合并成 4 位的 ch_mask 总线，接 APU 的 ch_mask 入口。（分线器在 Logisim 里是双向的：从「分」侧驱动、在「合」侧读出，就是合并器——这是标准接法。）

实时显示：在 CH0…CH3 四根线上各挂一个 探针（Probe）（基数（Radix）=二进制），即可看到四个声道当前的 使能（enable）状态；停止/某路关掉时对应探针变 0。
已验证参照：top_pureAI.circ 已按此实装（4 个 1 位开关 → 合并分线器 → ch_mask，外加 4 个探针），无界面 --tty stats 加载通过、位宽与 APU ch_mask 对齐无灰线。（headless 自检模式下这里仍退回 0xF 常量，因为无界面无法拨开关。）

和「暂停 / 停止」联动：暂停、停止都要静音

只门控时钟（C1）会让 CPU 冻结、但 APU 仍把最后一个音一直「嗡」住，不像暂停。停止（reset）更要命：reset 只复位 CPU（PC/busy/寄存器），根本没接到 APU（APU 没有 reset 输入口），所以 APU 内部 RegFile 里上个音的频率/reg9 全留着、声道还开着 → 长响。

统一修法：先算一个总开关 active = run 与 (非 stop)（一个 非门（NOT Gate） + 一个 与门（AND Gate）），再让每个声道位 = 对应开关 与 active（4 个 与门（AND Gate））。于是暂停（run=0）或停止（stop=1）任一发生，ch_mask 都被强制压成 0000 ⇒ 全部静音；恢复时按开关值还原。把那 4 个探针接在相与之后的线上，暂停/停止时探针一起变 0，直观反映"现在没在响"。（切歌 next/prev 走的是另一条 TRKCLK→reset，不算进 stop，所以切歌仍带声音、从新曲头干净开始。开关引脚是输入、值由你拨，逻辑上无法回写引脚本身，所以是"门控其效果"而非真把引脚清零；效果等同。）

★ 关键陷阱：光把 ch_mask 压成 0 还不够——暂停会"嗡住"

apu.circ 里每个声道的蜂鸣器使能不是直接吃 ch_mask，而是先进一个寄存器（Register）ENABLE_R，由 frame_clk 打拍采样（ch_mask位 → ENABLE_R.D →[frame_clk]→ ENABLE_R.Q → 蜂鸣器使能）。也就是说静音只在"frame_clk 来一拍、把 ch_mask=0 锁进去"那一刻才生效。而暂停=门控 CPU 时钟（C1），frame_clk（来自 frame_commit）也跟着停了 ⇒ ENABLE_R 永远锁不进那个 0 ⇒ 上个音一直嗡（停止同理：若此时时钟也停了）。

根因修法：暂停/停止时让 frame_clk 仍然在跳。给 APU 的 frame_clk 接 frame_commit 或 (母时钟 mclk 与 非active)（一个 非门 + 一个 与门 + 一个 或门）：播放时（active=1）=frame_commit（按帧干净采样，音质不变）；暂停/停止时（active=0）改吃自由跑的 mclk，于是 APU 仍每拍采样、把 ch_mask=0 锁进 ENABLE_R ⇒ 真静音；而 CPU/PC 仍被冻结 ⇒ 曲子停在原位。（此时 CPU 不在写寄存器，mclk 重复采样的是稳定值，无毛刺。）已在 top_pureAI.circ 实装并无界面仿真验证：暂停时 frame_clk 随 mclk 持续 0/1 翻转。

5LWAIT 长等待 兼容 · 可选

里程碑 3 的 WAIT 只处理 1 字、imm≤4095。LWAIT(opcode 0x2) 是 2 字指令：下一字才是 16 位等待数，用于 >4095 拍的长静音。track00 不含它(已核对)，只有个别长静音曲才需要。

实现 = 给 WAIT 状态机加一个「取操作数」小步骤：busy==0 且 opcode==0x2 时，先让 PC+1 去取下一字、置一个 lwait_fetch 标志；下一拍把 ROM[PC](那 16 位数) 装进 WAIT 计数器、置 busy、清 lwait_fetch；倒数完照常 PC+1(此时已自然跨过两字)。本质是再加一个一拍的状态，原理与 §里程碑 3 的 busy 机一致。

6里程碑 4 验收清单

  • 必做 ALU 接进 WAIT：tempo_mode 切 正常/慢/快 时，真实曲子的节奏随之变(快档不卡死)；加/减/钳位用 --test-vector 验证通过。
  • 必做 R0–R3 + LOADI/ADD/SUB/WRITEREG + 读/写口 + 写使能译码搭好；demo 程序现场显示 R2=R0+R1=8(改 SUB 得 2)。
  • 加分 控制面板：播放/暂停冻结、停止复位静音、上/下一首切 ROM（按实际曲数，4 首=2 位曲号+4:1 MUX）。
  • 加分 4 个声道开关（ch_mask）能单独开/关 Pulse1/Pulse2/Triangle/Noise，4 个 探针（Probe） 实时显示各路使能；现场单独静音某一路可闻、对应探针变 0。
  • 兼容（若做）LWAIT 2 字长等待正确。
  • 全程无红线 / 无灰线；top.circ 用 加载库 正确组装 cpu+apu；已存盘并备份 *_m4.circ。

7收尾 & 答辩要点

做完里程碑 4，整台播放器规格完整：五段 CPU + ALU 可控加减 + 寄存器 + APU 出声 + 控制面板 + 多曲。答辩时三条主线最稳：

  • 「ALU 不是摆设」：现场切 慢/快，曲子节奏当场变——因为每条 WAIT 都过加/减器(§2)；再用 demo 程序展示 R2=R0±R1(§3)。这正面回应老师「可控加减运算」。
  • 「五段单周期」：IF/ID/EX/MEM/WB 五阶段、非流水线(已向老师确认)；唯一的多周期处理是 WAIT 停顿 busy 状态机(里程碑 3 §4)。
  • 「真 NES 音乐」：从 NSF 离线转码成指令 ROM，CPU 读谱、APU 用真实 Hz 发声、节奏靠 WAIT+时钟分频(里程碑 1–3)。
