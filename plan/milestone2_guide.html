里程碑 2：让寄存器来驱动声音 —— 真正的单 Pulse 通道 里程碑 1 你用手直接喂 timer 让 Buzzer 出声。本里程碑把“手喂”换成“写寄存器”：搭一个寄存器前端（译码器 + 寄存器组）+ 一个可复用的 Channel_Pulse 子电路，从 $4000/$4002/$4003/$4015 解码出音高 / 音量 / 占空比 / 使能。做完你会得到一个用 NES 寄存器协议驱动的、可复用的单通道，并顺手把它扩成 4 路、封进 APU 子电路——这正是接 CPU 前的最后一块拼图。

里程碑 1 的结论（已实测，作为本里程碑的地基）
  • 频率表 + 蜂鸣器（Buzzer）链路音高正确：freq_pulse[253] = 440Hz = A4，手喂 timer 能弹音阶。
  • 蜂鸣器（Buzzer）扛得住频率变化（用分线器（Splitter）取计数器（Counter）高位分频后，扫频流畅）。
  • 核心原理（本里程碑要继续遵守）：蜂鸣器（Buzzer）每次输入变化都会重建音频缓冲，所以绝不能让高速时钟（Clock）每一拍都改它的输入。换音率应当压在音乐帧率 ~60Hz。本里程碑会专门加一道“按帧采样”门控来落实这一点（见 §4.5）。

0这一里程碑要搭什么

从“手喂 timer”到“写寄存器”

真实 NES 的 CPU 不会直接给声卡一个频率，它往寄存器里写字节：把音高拆成 $4002(低 8 位)+$4003(高 3 位)，把音量/占空比塞进 $4000，用 $4015 开关声道。APU 再把这些寄存器值“翻译”成频率/音量喂给发声单元。本里程碑就是在 Logisim 里复刻这套“寄存器 → 翻译 → 蜂鸣器（Buzzer）”的链路。

两个新子电路（都建在 apu.circ 里）
apu.circ
├── BuzzerStress      （里程碑 1，留作回归测试，不动）
├── PulseOneShot      （里程碑 1，留作对照，不动）
├── RegFile      ★新建：寄存器前端 = 译码器(Decoder) + 4 个寄存器(Register)
│                       对外像 CPU 的写口：reg_id(4) / value(8) / WR(1) 进，寄存器值出
└── Channel_Pulse ★新建：可复用的“一路 Pulse 通道”
                        = timer 拼接 + freq_pulse ROM + 音量/占空比/使能 + 蜂鸣器(Buzzer)
   （§5 再把 Channel_Pulse 复制/改造成 Pulse2 / Triangle / Noise，最后封一个 APU 顶层）

分两层的好处：RegFile 是所有通道共享的“写总线”（CPU 以后只跟它打交道）；Channel_Pulse 是纯“翻译+发声”，不关心写入时序。这样 4 个通道能共用一套寄存器，答辩时也好讲：“写口就这 3 根线”。

1本里程碑新增部件速查

里程碑 1 用过的 蜂鸣器（Buzzer）/ ROM / 时钟（Clock）/ 计数器（Counter）/ 常量（Constant）/ 引脚（Pin）/ Splitter（分线器）不再赘述。下面是本里程碑新登场的三个部件，数值均依据 ref/logisim-evolution（v4.1.0）源码核实。

寄存器（Register）的脚怎么认

和里程碑 1 一样，嫌默认大符号难认就把 外观（Appearance） 换成 经典（Classic）。经典外观下寄存器（Register）= 一个方块：D 在左、Q 在右、底部带 ▷ 的是时钟、另一个小脚是写使能（en）。鼠标悬停在任意脚上会弹出 tooltip（如“输入：写使能（为 0 时忽略时钟触发）”），照提示接即可，别死记方位。

2寄存器前端 RegFile：译码器 + 寄存器组

先建“写口”。它接收 reg_id / value / WR，把 value 锁进 reg_id 选中的那个寄存器。本里程碑只需 Pulse1 用到的 4 个寄存器：reg0=$4000、reg2=$4002、reg3=$4003、reg9=$4015（声道使能；按计划 $4015 重映射到 reg_id 9）。

R1

新建子电路 RegFile

确认在编辑 apu.circ。项目 → 添加电路…（Project → Add Circuit…），名字 RegFile，双击进入。

R2

放 3 个输入 引脚（Pin）当“写口”

线路 → 引脚（Wiring → Pin），放三个，分别选中改属性：
• 类型（Type）=Input，数据位宽（Data Bits）=4，标签（Label）=reg_id
• 类型（Type）=Input，数据位宽（Data Bits）=8，标签（Label）=value
• 类型（Type）=Input，数据位宽（Data Bits）=1，标签（Label）=WR（这根当“写脉冲/时钟”用）

R3

放 译码器（Decoder），设 4 选择位 → 16 路

复用器 → 译码器（Plexers → Decoder）。选中它：选择位数（Select Bits）=4（输出变成 16 路）。把 reg_id 接到它的选择脚。
若它带“使能”脚（属性 启用使能（Include Enable?）=是）：要么把该属性改成 否，要么给使能脚接一个 值=1 的 常量（Constant），否则整个译码器被禁用、所有输出为 0。

R4

放 4 个 寄存器（Register），各 8 位

存储器 → 寄存器（Memory → Register） 放 4 个，每个 数据位宽（Data Bits）=8，触发方式（Trigger） 保持上升沿。建议各贴 标签（Label）：reg0、reg2、reg3、reg9。

R5

接线：value→D，WR→时钟，译码器输出→写使能

对这 4 个寄存器：
• value(8-bit) 同时接到每个寄存器的 D（数据入）。一根总线分叉到 4 处即可。
• WR(1-bit) 同时接到每个寄存器的时钟脚(▷)。
• 译码器（Decoder）的第 0 路输出 → reg0 的写使能；第 2 路 → reg2；第 3 路 → reg3；第 9 路 → reg9。（其余 12 路输出悬空不接。）
• 每个寄存器（Register）的清除脚接一个 1 位、值为 0 的 常量（Constant）。不要让清除脚飘着；源码里 CLR==1 会异步清零，接 0 最稳。

为什么这样接就对

WR 上升沿到来时，4 个寄存器都收到时钟边沿，但只有写使能=1 的那个会真正把 value 锁进去。译码器保证“同一时刻最多一路使能=1”（reg_id 选中谁，谁的使能才是 1）。于是 reg_id=2, value=0xFD, 点一下 WR ⇒ 只有 reg2 变成 0xFD。这正是 CPU WRITE reg,val 的硬件含义。

R6

放 4 个输出 引脚（Pin），把寄存器值引出去

再放 4 个 引脚（Pin），类型（Type）=Output、数据位宽（Data Bits）=8、标签 reg0/reg2/reg3/reg9，分别接到对应寄存器的 Q。这样 RegFile 作为子电路被实例化时，外面就能拿到这 4 个寄存器的当前值。保存文件。

线太多？用 隧道（Tunnel）

value 要分叉到 4 个 D、WR 要分叉到 4 个时钟——画起来容易乱。可用 线路 → 隧道（Wiring → Tunnel）：给一根线起个名字（如 VAL），别处放同名 隧道（Tunnel）就等于连上了，不用真的拉线。§5 扩到 4 通道时这招能救命。

3Channel_Pulse 子电路：把寄存器值翻译成声音

这是本里程碑的主菜。输入是 4 个寄存器值（reg0/reg2/reg3 + 本通道使能位），输出是蜂鸣器（Buzzer）的发声。结构和里程碑 1 的 PulseOneShot 一脉相承，只是 timer 不再手喂、而是从 reg2/reg3 拼出来，音量/占空比/使能也都从寄存器解出来。

C1

新建子电路 Channel_Pulse，放好 4 个输入 引脚（Pin）

项目 → 添加电路…（Project → Add Circuit…） → Channel_Pulse，进入。放 4 个输入 引脚（Pin）：reg0(8)、reg2(8)、reg3(8)、enable(1)。（enable 是本通道的使能位，等会儿在外面接 reg9 的第 0 位。）

3.1 拼出 11 位 timer（本里程碑的关键动作）

C2

第一个 Splitter（分线器）：从 reg3 里取出低 3 位

放一个 Splitter（分线器）。属性：输入位宽（Bit Width In）=8、分支数（Fan Out）=2。
选中后属性表会列出 Bit 0 … Bit 7 各归到哪个分支：把 Bit 0、1、2 设为分支 0（这就是我们要的低 3 位），Bit 3–7 设为分支 1（丢弃不接）。
把 reg3 接到 Splitter 的合并端（那一根 8 位侧）；分支 0 引出来就是 reg3[2:0]（3 位）。

C3

第二个 Splitter（分线器）：把 reg2(8) + reg3低3(3) 拼成 timer(11)

再放一个 Splitter（分线器），这次当“合并器”用。属性：输入位宽（Bit Width In）=11、分支数（Fan Out）=2。位分配：Bit 0–7 → 分支 0（低 8 位），Bit 8–10 → 分支 1（高 3 位）。
接线：reg2(8 位) → 分支 0；上一步的 reg3[2:0](3 位) → 分支 1。合并端引出的 11 位总线就是 timer = (reg3[2:0]<<8) | reg2。

位宽必须严丝合缝

分支 0 必须正好 8 位、分支 1 正好 3 位，合起来 11 位。线是黑色（多位总线）才对，灰色 = 位宽没对上（里程碑 1 讲过）。Splitter 的方向/朝向若让接线别扭，调 朝向（Facing） 即可。

3.2 timer → 频率（复用里程碑 1 的频率表）

C4

放 freq_pulse ROM

存储器 → ROM（Memory → ROM）。务必先设 地址位宽（Address Bit Width）=11、数据位宽（Data Bit Width）=14，再右键 加载映像…（Load Image…） → script/out/freq_pulse.txt（对话框那行应是 decoded 2048 of 2048 words, 14 bits each；若显示 256 / 8 就是位宽没设，见里程碑 1 红框）。
timer(11) → ROM 的 A。

3.3 音量 / 占空比 / 使能（从 reg0、enable 解出来）

C5

第三个 Splitter（分线器）：把 reg0 拆成音量位和占空比位

放一个 Splitter（分线器）：输入位宽（Bit Width In）=8、分支数（Fan Out）=3。位分配：Bit 0–3 → 分支 0（音量，4 位）、Bit 4–5 → 分支 1（丢弃）、Bit 6–7 → 分支 2（占空比，2 位）。reg0 接合并端。

C6

音量：reg0[3:0] → 蜂鸣器（Buzzer）的 VOL

放 蜂鸣器（Buzzer）（输入/输出（扩展）→ 蜂鸣器（Input/Output Extra → Buzzer）），波形（Waveform）=Square（方波）。把它的 音量位宽（Volume Width） 改成 4，然后把 reg0 分线器的分支 0（4 位音量）接到 VOL 脚。这样寄存器里 0–15 的音量直接生效（NES Pulse 音量本就是 4 位）。

C7

占空比：reg0[7:6] → 4 项查找 → PW 可后补

NES 占空比 2 位选 4 档 {12.5%,25%,50%,75%}，对应蜂鸣器（Buzzer）8 位 PW 的 {32,64,128,192}={0x20,0x40,0x80,0xc0}。做一个 4 项查找 ROM：
放一个 ROM，地址位宽=2、数据位宽=8，右键 编辑内容…（Edit Contents…） 填 4 个值 20 40 80 c0。把 reg0 分线器的分支 2（2 位占空比）接 ROM 的 A，ROM 的 D(8) 接蜂鸣器（Buzzer）的 PW 脚。
想先简单点：这一步可暂时跳过——PW 直接接一个 值=0x80(50%) 的 8 位 常量（Constant），等整条链路出声后再回来接占空比。

C8

使能：enable → 蜂鸣器（Buzzer）的 ENABLE

把输入 引脚（Pin）enable(1 位) 接到蜂鸣器（Buzzer）的 ENABLE 脚。enable=1 出声、=0 静音——这就是 $4015 声道开关的硬件落点。

C9

FREQ 接好，存盘

把 freq_pulse ROM 的 D(14 位) 接到蜂鸣器（Buzzer）的 FREQ 脚（两端都 14 位，黑线）。至此 Channel_Pulse 内部完工：4 个输入 引脚（Pin）进，一个蜂鸣器（Buzzer）出。保存文件。

判定标准（本里程碑的“出声”门槛）
  • 音高对：写 reg2/reg3 改 timer，验证“数值越小、音越高”（参考里程碑 1 的表：253→A4、190→D5、126→A5、63→A6；这里 reg2=低 8 位，timer<256 时 reg3 都写 0）。
  • 音量可控：把 reg0 改成 88(音量 8)、81(音量 1)，音量随之变小；80(音量 0) 几乎无声。
  • 占空比可控（若接了 C7）：reg0 写 0F/4F/8F/CF ⇒ 12.5/25/50/75%，音色（明亮↔圆润）有变化。
  • 使能可控：reg9 写 00 立即静音、写 01 复响。

四条都过 = 寄存器协议 → 声音这条链路打通。这是接 CPU 的前提（CPU 无非就是自动地、按曲谱去做你刚才手动做的那串“写 reg_id/value + WR”）。

4.5★ 按帧采样门控：自动扫表/接 CPU 前的保护

为什么现在就要加这道门

§4 你是用手慢慢写寄存器，所以蜂鸣器（Buzzer）输入变化很慢、毫无压力。但一旦用计数器或 CPU 自动写寄存器，上游可能在很短时间内连续改 reg2/reg3/reg0/reg9。回顾里程碑 1 的命门：蜂鸣器（Buzzer）每次输入变化都会置 updateRequired=true 并重建音频缓冲（见 ref/.../Buzzer.java 的 propagate() 和后台线程）。如果把高速变化直接喂进去，必然爆音/卡死。
保护办法是把送进蜂鸣器（Buzzer）的值按音乐帧率（约 60Hz）采样保持：无论上游变多快，蜂鸣器（Buzzer）每秒只被更新几十次。它是本里程碑从“手测”过渡到“自动化”的保护层；等接 CPU 时，最好把这根采样脉冲换成 CPU 的“本帧写完/进入 WAIT”提交信号，而不是永远用一个自由跑的慢时钟。

G1

给 Channel_Pulse 增加一个输入 frame_clk

回到 Channel_Pulse，加一个输入 引脚（Pin）frame_clk(1 位)。它是“每帧一拍”的慢时钟，由外面送进来（下一步生成）。

G2

在蜂鸣器（Buzzer）前插一排采样 寄存器（Register）

把原本“直连蜂鸣器（Buzzer）”的 4 路信号，各经过一个 寄存器（Register） 再进蜂鸣器（Buzzer）：
• FREQ：14 位 寄存器（Register）；  • VOL：4 位；  • PW：8 位；  • ENABLE：1 位。
每个采样 寄存器（Register）：D 接原信号、Q 接蜂鸣器（Buzzer）对应脚、时钟脚统一接 frame_clk、写使能接 1 位常量 1、清除脚接 1 位常量 0。于是蜂鸣器（Buzzer）只在 frame_clk 上升沿才看到新值。

至少把 FREQ 锁住

嫌 4 个寄存器多，最低限度给 FREQ 加采样保持即可（频率变化是爆音主因）。但 4 路都锁最稳，推荐全锁。

G3

在测试台生成 ~60Hz 的 frame_clk（复用里程碑 1 的分频招式）

回到 main。放 时钟（Clock） + 计数器（Counter），时钟（Clock）接计数器（Counter）；用 Splitter（分线器）取计数器（Counter）的某一位当 frame_clk，接到 Channel_Pulse.frame_clk。
这里用的是 Logisim 左上角显示的完整周期/s，不是菜单里的半周期 tick 数；状态栏源码 TickCounter.java 会把菜单频率除以 2 来显示。采样 寄存器（Register）看的是 frame_clk 的上升沿，所以若取 Counter[n]，采样率约为 左上角显示值 / 2^(n+1)。推荐直接照表接：

自动节拍频率（Auto-Tick Frequency）左上角约显示取哪一位作 frame_clk采样率
1000Hz500HzCounter[2]500/8 = 62.5Hz
2kHz1000HzCounter[3]1000/16 = 62.5Hz
4kHz2000HzCounter[4]2000/32 = 62.5Hz

建议先用 2kHz + Counter[3]。如果界面里 2kHz 不卡，就不要再调。这个接法和里程碑 1 的“取高位扫表”是同一个思想，只是这里取的是一个单 bit 慢脉冲，而不是 11 位 ROM 地址。

G4

开 自动节拍（Auto-Tick）重测

这次必须勾选 仿真 → 启用自动节拍（Simulate → Auto-Tick Enabled），并按上表设置 仿真 → 自动节拍频率（Simulate → Auto-Tick Frequency）（否则 frame_clk 不跳、采样 寄存器（Register） 永不更新、蜂鸣器（Buzzer）不出声）。重做 §4 的写寄存器操作：声音应当照常正确，且改 timer/音量时干净不爆。

接了 CPU 之后这道门会怎样

下一里程碑 CPU 用 WAIT 让 PC 停顿，把写寄存器的节奏天然压到音乐帧率（~60Hz）。但同一帧开头可能连续写好几个寄存器（比如先写 timer 低位、再写高位、再写音量），所以自由跑的 frame_clk 有概率采到“半更新”的组合。最终接 CPU 时，最佳做法是让 CPU 在一帧写完、开始执行 WAIT 时给 APU 一个 frame_commit 脉冲，并把它接到这里的 frame_clk。在本里程碑手测阶段，先用上面的分频慢时钟即可。

5扩展成 4 通道 + 封装 APU 子电路

单通道验证通过后，剩下的基本是复制 + 改几个参数。先把 RegFile 补全到 4 通道要用的寄存器，再造另外 3 个通道子电路，最后封一个 APU 顶层。

P1

把 RegFile 补全到所需寄存器

回 RegFile，按同样方式补上其余寄存器并引出 Q：Pulse2 用 reg4/reg6/reg7、Triangle 用 reg8/reg10/reg11、Noise 用 reg12/reg14（reg9 已有，作全局使能）。译码器（Decoder）对应输出路接到各自寄存器的写使能即可。这里强烈建议用 隧道（Tunnel）按名字布线，否则线会爆炸。

P2

造 Channel_Triangle / Channel_Noise（照 Channel_Pulse 改）

复制 Channel_Pulse 的思路，按下表改动：

通道波形（Waveform）频率来源音量 / 占空比使能位
Channel_Pulse(P1)方波 Squarereg2+reg3低3 → freq_pulse(addr11)VOL=reg0[3:0]；PW=reg0[7:6]查表reg9[0]
Pulse2（再实例化一个 Channel_Pulse）方波 Squarereg6+reg7低3 → freq_pulseVOL=reg4[3:0]；PW=reg4[7:6]reg9[1]
Channel_Triangle三角波 Trianglereg10+reg11低3 → freq_triangle(addr11)无音量寄存器→VOL 接常量；无占空比reg9[2]
Channel_Noise白噪声 White noisereg14[3:0] → freq_noise(addr4)VOL=reg12[3:0]；无占空比reg9[3]

注意 Noise 的频率表是 16 项、地址 4 位（freq_noise.txt，地址位宽=4、数据位宽=14），且 timer 不用拼接，直接拿 reg14 低 4 位当索引。Triangle 没有音量寄存器，VOL 给个固定值即可（响不响由使能管）。三者都保留 §4.5 的 frame_clk 采样门。

P3

封 APU 顶层子电路

新建子电路 APU：对外只暴露 CPU 关心的写口 reg_id(4) / value(8) / WR(1) + frame_clk(1)（以后可由 CPU 的 frame_commit 驱动；外加可选的总静音/复位）。内部：一个 RegFile + 4 个通道子电路。
用一个 Splitter（分线器）把 reg9 拆成 4 个使能位，分别接 4 个通道的 enable（reg9[0..3] = P1/P2/Tri/Noise）。各通道的寄存器值从 RegFile 的对应输出接入（隧道（Tunnel）布线）。
4 个蜂鸣器（Buzzer）并联发声——系统声卡会自动混音（里程碑 1 已验证多蜂鸣器（Buzzer）可同响）。

4 路是目标，但单 Pulse 才是本里程碑硬门槛

源码和详细计划都支持 4 个蜂鸣器（Buzzer）并联，但它们各自开音频线程/Clip，真实机器上仍可能受声卡或 Java 音频后端影响。若 4 路同时开有卡顿，先保 Channel_Pulse + APU 写口正确，再降级到 2 Pulse 或 2 Pulse + Triangle 演示；不要为了 4 路音色拖垮后面的 CPU 里程碑。

手动测 4 路和声

在测试台实例化 APU，按 §4.5 的表开启 自动节拍（Auto-Tick）并接好 frame_clk。手写几组寄存器：给 Pulse1/Pulse2/Triangle 各写一个不同 timer、reg9=0x07（开前三路）——应当听到三个音叠成的和弦。再写 reg9=0x00 应当整体静音。能出和声 = APU 骨架成立；若多路卡顿，记录为性能风险，先保单 Pulse 正确。

6里程碑 2 验收清单

  • RegFile 子电路：reg_id/value/WR 写口 + 译码器（Decoder）+ 寄存器（Register）组，写哪个寄存器只动哪个（手测确认）。
  • Channel_Pulse 子电路：用两个 Splitter（分线器）把 reg2 + reg3低3 拼成 11 位 timer，经 freq_pulse ROM 出 A4。
  • 音量（reg0[3:0]）、占空比（reg0[7:6]，可后补）、使能（reg9[0]）均可控。
  • §4.5 的 frame_clk 按帧采样门已加：按推荐表取 Counter 位时声音干净不爆；并知道接 CPU 后应优先改接 frame_commit。
  • （扩展）4 个通道 + APU 顶层封好，能手动写出三音和弦、能整体静音。
  • 全程无红线 / 无灰线；已存盘并备份 apu_m2.circ。
做完这一步，你已经具备了什么

一个用 NES 寄存器协议驱动的 APU：外界只要按 reg_id/value/WR 往里写字节，它就发声。这意味着——只要再有一个会自动“吐 WRITE/WAIT/END”的东西，音乐就播起来了。那个东西就是 CPU，正是下一里程碑。

7下一里程碑预告

里程碑 3 = 搭 CPU（cpu.circ）并接上本里程碑的 APU：单周期五段（IF/ID/EX/MEM/WB）+ WAIT 停顿状态机 + 取指/译码 WRITE/END，把 track00.txt 那条 16 位指令流跑起来，自动产生你刚才手动做的那串“写 reg_id/value + WR”。届时 CPU 的 WAIT 停顿会接管音乐帧率，§4.5 的采样门退居二线。唯一的正确性陷阱是 WAIT 的 busy 状态机（见 plan/detailed_plan.html §5），到时单开一份文档细讲。
