CPU Explanation · Single-Cycle Five Logical Stages

我们的 CPU 能否满足“单周期五段 CPU”的设计要求？

结论：可以，但要准确表述为 单周期五段 CPU + WAIT 多周期扩展。普通指令在一个时钟周期内走完 IF / ID / EX / MEM / WB；为了音乐节奏，WAIT 指令会故意暂停 PC 若干拍。

一句话结论

当前 cpu.circ 不是通用 MIPS/RISC CPU，而是为 NSF 播放器定制的 16-bit ISA CPU。 它仍然具备单周期 CPU 的五个逻辑阶段：取指、译码、执行、访存、写回。 区别在于：它的 MEM 阶段主要访问 APU 内存映射寄存器，而不是普通数据 RAM。

1. 标准五段如何对应到本项目 CPU

标准阶段 标准含义 本项目中的实现 对应信号 / 部件
IF
取指令		 根据 PC 地址从指令存储器取出一条指令。 CPU 输出 pcbus，顶层 top.circ 用它访问曲目 ROM，再把 16-bit 指令送回 instr。 PC 计数器（Counter）、pcbus、instr、曲目 ROM
ID
指令译码		 由指令生成控制信号，明确这条指令要做什么。 固定 16-bit 指令用 Splitter（分线器）拆出 opcode、reg_id、value、imm 等字段，再产生 IS_WAIT、IS_ADD、IS_SUB 等控制信号。 Splitter（分线器）、译码器（Decoder）、控制信号
EX
执行		 对操作数进行运算。 ALU 完成 ADD/SUB；WAIT 装载路径计算 imm ± tempo_delta；WAIT 状态机判断是否暂停 PC。 加法器（Adder）、减法器（Subtractor）、比较器（Comparator）、多路复用器（Multiplexer）、WAITCNT、busy
MEM
存储器访问		 将数据写入存储器，或从存储器读出数据。 WRITE/WRITEREG 在这一阶段访问 APU 写口，把 reg_id 和 value 写入 APU RegFile。这里属于内存映射 I/O。 reg_id、VALUE_OUT、wr、frame_commit、APU RegFile
WB
写回		 将运算结果写回寄存器。 更新 PC；把 ALU 结果写回 R0-R3；更新 WAITCNT 和 busy 等状态元件。 R0-R3 寄存器（Register）、PC 计数器（Counter）、WAITCNT、busy

2. 为什么 MEM 阶段写 APU 也算“存储器访问”

标准 CPU 课程里，MEM 阶段常被讲成“访问 RAM”。但真实 CPU 里，MEM 阶段也可以访问 内存映射 I/O。NES 2A03 正是这种设计：CPU 写 $4000-$4017 不是在写普通 RAM，而是在写 APU 的硬件寄存器。 

CPU 执行 WRITE
  -> MEM 阶段输出 reg_id / value / wr
  -> APU RegFile 被写入
  -> APU 通道把寄存器解释成频率、音量、占空比、使能
  -> 蜂鸣器（Buzzer）发声
答辩时的推荐说法

本 CPU 的 MEM 阶段访问的不是普通数据 RAM，而是 APU 的内存映射寄存器组。 这和真实 NES 的 CPU/APU 交互方式一致：CPU 通过写固定地址控制声音硬件。

3. 唯一要主动说明：WAIT 是多周期扩展

如果严格说“所有指令都一个周期”，那不准确。我们的 WRITE、ADD、 SUB、WRITEREG、END 等普通指令可以按单周期完成； 但 WAIT 是播放器为了节奏控制专门加入的多周期指令。

指令类型执行方式原因
WRITE / ADD / SUB / WRITEREG 一个时钟周期内走完 IF/ID/EX/MEM/WB。 符合单周期 CPU 的基本要求。
WAIT n 第一拍装载 WAITCNT，随后 busy 保持 PC 不变，倒数到 0 再放行。 音乐必须有时间间隔；否则 ROM 会被高速扫完，节奏不存在。
不要这样说

“我们的所有指令都严格一个周期完成。”

应该这样说

“我们的 CPU 是单周期五段结构，普通指令一个周期完成；为了音乐播放，额外实现了 WAIT 多周期停顿指令，让 PC 按节奏暂停。”

4. 对照老师给出的五段定义

老师定义本项目是否满足说明
IF：根据 PC 取指令满足pcbus 输出地址，外部 ROM 返回 instr。
ID：生成控制信号满足Splitter（分线器）拆指令字段，译码出 IS_WAIT、IS_WRITE 等信号。
EX：运算操作数满足ALU、WAIT 调速路径、比较器（Comparator）均属于执行阶段。
MEM：访问存储器满足访问 APU 内存映射寄存器；这是 I/O 型存储器访问。
WB：写回寄存器满足写回 R0-R3、更新 PC / WAITCNT / busy 等状态。

5. 最终答辩用短版

我们的 CPU 是单周期五段 CPU，但不是通用 MIPS，而是面向音乐播放器的定制 ISA。
IF 阶段由 PC 通过 pcbus 取外部曲目 ROM 的 instr；
ID 阶段用 Splitter 拆 opcode 和操作数字段；
EX 阶段执行 ALU、WAIT 状态机和调速计算；
MEM 阶段写 APU 的内存映射寄存器；
WB 阶段更新 PC、R0-R3 和 WAIT 状态。

普通指令一个周期完成；WAIT 是为了音乐节奏额外加入的多周期停顿指令。
所以准确说法是：单周期五段 CPU + WAIT 多周期扩展。
